# KanonOS – MVP Roadmap

**Version:** 1.0  
**Status:** Active  
**Last Updated:** 2026-07-08

---

# Purpose

This roadmap defines the development sequence of the KanonOS MVP.

The objective is to deliver value early through small, functional iterations.

Each sprint should result in a usable product increment.

---

# MVP Goal

A coach should be able to replace paper, Word and Excel during an entire training week using KanonOS.

---

# MVP Principles

Every sprint must:

- Deliver a usable feature.
- Solve a real coaching problem.
- Build on previous work.
- Avoid unnecessary complexity.

---

# Sprint 1 – Foundation

Objective:

Build the application structure.

Deliverables:

- Authentication (Demo Mode)
- Sidebar navigation
- App layout
- Coach Workspace
- Design System
- Routing
- Demo data

Status:

Completed

---

# Sprint 2 – Teams

Objective:

Create the coach's working context.

Deliverables:

- Team list
- Team details
- Player list
- Coach assignment
- Season selection

Success:

The coach can organise and navigate between teams.

---

# Sprint 3 – Weekly Planning

Objective:

Plan the coaching week.

Deliverables:

- Weekly objective
- Training calendar
- Create training session
- Weekly notes

Success:

The coach no longer needs Word or paper to plan the week.

---

# Sprint 4 – Training Session

Objective:

Build and execute a training session.

Deliverables:

- Session objective
- Exercises
- Materials
- Timeline
- Coach notes
- Training Mode

Success:

The coach can use KanonOS during training.

---

# Sprint 5 – Training Review

Objective:

Capture what happened.

Deliverables:

- Attendance
- Session rating
- Coach reflection
- Improvements
- Session completed

Success:

Every training session generates historical knowledge.

---

# Sprint 6 – History

Objective:

Create memory.

Deliverables:

- Previous sessions
- Attendance history
- Weekly history
- Team timeline

Success:

The coach can review previous work without external documents.

---

# Post-MVP

The following modules are intentionally excluded from the MVP.

Planning

- Annual Planning
- Mesocycles
- Microcycles

Performance

- Player Evaluation
- Athlete Passport
- Wellness Questionnaires

Communication

- Parent Portal
- Athlete Portal
- Notifications

Management

- Coordinator Dashboard
- Club Management

Artificial Intelligence

- AI Planning Assistant
- Automatic Reports
- Exercise Suggestions

Integrations

- Google Calendar
- Outlook
- Hudl
- Spiideo
- Federation APIs

---

# Definition of Done

A sprint is complete when:

- The feature works.
- The UX is consistent.
- The feature is tested.
- Documentation is updated.
- The coach can use it without explanation.

---

# Success Metrics

The MVP is considered successful when coaches can:

- Create a team.
- Plan an entire week.
- Prepare a training session.
- Use the session during practice.
- Record attendance.
- Review previous sessions.

Without using external tools.
